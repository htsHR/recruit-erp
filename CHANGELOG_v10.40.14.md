# Recruit ERP v10.40.14 HOME_DEV · EMPLOYEE_ORG_ENRICHMENT

## 기준 버전
- v10.40.13 EMPLOYEE_UI_DETAIL_FINAL

## 주요 변경
- 사원명부 상단에 `엑셀 조직정보 반영` 기능 추가
- 엑셀에서 추출한 조직정보 전용 JSON을 사번 기준으로 기존 사원과 비교
- 반영 대상 필드: 팀 / 그룹 / 제품 / 파트
- 사번 일치 후 성명까지 동일한 사원만 반영 가능
- 변경 가능 / 동일 / 이름 불일치 / 사번 없음 / 오류 분류
- 변경 가능한 사원만 기본 선택하고 사용자가 선택한 인원만 반영
- 엑셀의 빈칸과 `-` 값은 기존 조직정보를 삭제하지 않음
- 이름 불일치, 사번 미매칭, 중복 사번은 적용 차단
- 반영 전 ERP 전체 JSON 안전 백업 자동 다운로드
- 반영 후 로컬 저장 및 로그인 상태의 Supabase 저장 자동 시도
- 회사 모드에서는 파일 업로드와 일괄 반영 차단
- 주민번호·연락처·주소·이메일이 포함된 파일은 검사 단계에서 거부

## 엑셀 원본 처리 기준
- `(주)에이치티솔루션 사원명부` 685명
- `휴직자 명단` 7명
- `퇴직자 현황` 689명
- 총 1,381명의 사번·성명·팀·그룹·제품·파트만 추출
- 퇴직자 현황의 과거 열 배치 차이는 팀/그룹 값 패턴으로 보정
- `FLASH`는 `Flash`, `기술1그룹`은 `기술 1그룹`으로 표준화

## 사용 순서
1. 집 모드에서 회사 전체 JSON을 복원합니다.
2. v10.40.13의 `supabase_migration_v10.40.13_employee_fields.sql`을 아직 실행하지 않았다면 Supabase SQL Editor에서 1회 실행합니다.
3. 사원명부의 `엑셀 조직정보 반영`을 누릅니다.
4. 별도로 제공된 `명단_조직정보_반영용_v1.json`을 선택합니다.
5. 비교 결과와 변경 항목을 검토하고 확인 체크 후 반영합니다.
6. 반영 완료 후 전체 JSON 백업과 Supabase 저장 상태를 확인합니다.

## 개인정보 주의
- 조직정보 전용 JSON은 이름과 사번이 포함된 내부 업무 파일입니다.
- GitHub, Vercel 또는 공개 저장소에 업로드하지 마세요.
- 배포용 ZIP에는 해당 JSON을 포함하지 않았습니다.
