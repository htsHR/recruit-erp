## 주요 변경
- 회사 모드에서 JSON 업로드·검사·병합·전체교체·복원 코드 실행 차단
- 퇴근 전 점검: 저장소 상태, 핵심 데이터 건수, 오늘 전체 백업, 백업 이후 추가·수정·삭제 확인
- 전체 백업 시 데이터별 무결성 서명과 백업 기준 스냅샷 저장
- 마지막 백업 이후 데이터별 추가·수정·삭제 건수 추적
- 집 모드 JSON 검사 강화: 버전·배열·기록 건수·실제 건수·무결성 검증
- 현재 데이터와 파일 데이터의 신규·변경·파일에 없음 건수 비교
- 지원자만 있는 구형 배열 JSON과 구형 객체 JSON 구분 지원
- 0건 교체 및 절반 이상 감소 위험 경고
- 병합·전체교체·전체복원 직전 현재 ERP 전체 안전 백업 자동 다운로드
- 로컬 적용과 Supabase 저장을 분리하고 저장 결과를 화면에서 확인
- 병합 저장: 신규·수정 데이터 upsert 후 클라우드 건수 검증
- 전체 일치 저장: 모든 upsert 성공 후 로컬에 없는 클라우드 행만 정리하고 최종 건수 검증
- 직원 1,000행 제한을 피하기 위한 500행 페이지 조회 및 250행 단위 저장·삭제
- 지원자 전체 일치 저장 직전 기존 Supabase 스냅샷 생성 시도
