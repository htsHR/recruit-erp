# Recruit ERP v10.46.1 HOME_DEV
## DAILY_WORKFLOW_IMPROVEMENT

기준 버전: v10.46.0 APPLICANT_PROGRESS_HISTORY_PACK

## 변경 내용
- 기존 `오늘 할 일` 화면을 단일 처리 목록 중심으로 단순화
- 별도 입력 없이 기존 지원자 필드로 업무 자동 분류
  - 연락 필요
  - 연락기한 경과
  - 오늘 면접
  - 내일 면접
  - 면접 결과 미입력
  - 입사 예정(D-7)
  - 출근 확인 지연
  - 14일 이상 장기 미처리
- 한 지원자가 여러 조건에 걸릴 때 한 행에 사유를 합쳐 중복 표시 방지
- 요약 카드 및 업무 유형 필터 제공
- 이름·연락처·근무지·학교 검색 제공
- 목록에서 지원자 상세/수정 화면으로 바로 이동
- 기존 출근 지원자 → 사원명부 연결 기능 유지
- 기존 면접 명단표 인쇄 기능 유지
- 홈 화면의 연락 필요·기한 경과 집계에 `nextContactDate` 반영

## 데이터 영향
- 신규 저장 필드 없음
- 신규 localStorage 키 없음
- Supabase 마이그레이션 없음
- 지원자 상태 자동 변경 없음
- 기존 지원자 ID, 학교 연결, 사원 연결 유지
