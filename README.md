# Recruit ERP v10.36.6 HOME_DEV · LIST_UX

## 버전 목적

v10.36.5의 공통 레이아웃을 유지하면서, 실제 업무에서 가장 자주 사용하는 목록 화면의 가독성과 조작 동선을 개선한 UI·UX 전용 버전입니다.

## 이번 버전의 주요 개선

### 지원자 목록

- 데스크톱에서 상태·성명 등 핵심 열을 왼쪽에 고정
- 관리 열을 오른쪽에 고정해 가로 이동 중에도 작업 버튼 유지
- 행 클릭 또는 키보드 Enter로 지원자 상세보기 진입
- 모바일에서는 표를 카드형 목록으로 표시
- 모바일 기본 버튼을 `상세`, `수정`, `더보기`로 정리
- 복제·삭제 등 빈도가 낮거나 위험한 작업은 `더보기` 안으로 이동

### 사원명부

- 상태 탭에 전체·재직·퇴사·입사예정·휴직 인원수 표시
- 검색 결과 요약 문구 추가
- 행 클릭 또는 키보드 Enter로 직원 상세보기 진입
- 모바일 사원 목록을 카드형으로 표시
- 모바일 삭제 버튼을 `더보기` 안으로 이동
- 이름·사번·학교 검색창에서 Enter 검색 지원

### 협력학교 관리

- 학교 행 클릭 또는 키보드 Enter로 학교 상세보기 진입
- 모바일 학교 목록을 카드형으로 표시
- 학교 수정은 기본 노출, 삭제는 `더보기` 안으로 이동
- 모바일 목록의 고정 높이를 제거해 카드와 페이지네이션 겹침 방지

## 데이터 안전 범위

이번 버전은 목록 UI·UX만 개선했습니다.

- 지원자 데이터 구조 변경 없음
- 협력학교 데이터 구조 변경 없음
- 사원명부 데이터 구조 변경 없음
- JSON 형식 변경 없음
- LocalStorage 키 변경 없음
- Supabase 설정 및 저장 로직 변경 없음
- 기존 등록·수정·삭제·필터·페이지네이션 로직 유지

현재 주요 LocalStorage 키:

- `recruit_erp_applicants_stable`
- `recruit_erp_schools`
- `recruit_erp_employees`

압축을 푼 뒤 폴더 안의 `index.html`, `app.js`, `css/` 등 파일을 GitHub 저장소 최상단에 덮어쓰면 됩니다.

운영 데이터가 있는 환경에 적용하기 전에는 기존 v10.36.5 폴더와 JSON 백업을 별도로 보관하세요.
