# Recruit ERP v10.40.16

## JSON_TYPE_GUARD

- 백업센터에서 JSON 파일 유형을 먼저 판별합니다.
  - ERP 전체/부분 백업 JSON
  - 지원자 전용 JSON
  - 사원명부 전용 JSON
  - 협력학교/수동 일정 전용 JSON
  - 사원 조직정보 반영용 JSON
- 사원 조직정보 전용 파일을 백업센터에서 선택하면 병합·전체교체·전체복원 버튼을 생성하지 않습니다.
- 올바른 메뉴인 `사원명부 → 엑셀 조직정보 반영`을 안내하고 사원명부 이동 버튼을 표시합니다.
- 사원 조직정보 반영 메뉴에서도 ERP 백업·지원자·사원명부 파일을 구분해 잘못된 파일 적용을 차단합니다.
- 파일명, 크기, 파일 유형, 포함 데이터 요약, 버전, 생성일, 생성 환경을 표시합니다.

## EMPLOYEE_ORG_FILTER_FINAL

- 사원명부 검색에 팀·그룹·제품·파트 필터를 추가했습니다.
- 팀 → 그룹 → 제품 → 파트 순서로 선택지가 연동됩니다.
- 현재 결과 인원과 결과 내 팀·그룹·제품·파트 종류 수를 표시합니다.
- 목록의 소속은 팀을 첫 줄, `그룹 · 제품 · 파트`를 둘째 줄에 표시합니다.
- 현재 검색·조직 필터를 최대 12개까지 브라우저에 저장하고 다시 불러오거나 삭제할 수 있습니다.
- 저장 조건 키: `recruit_erp_employee_saved_filters_v1`

## 데이터 영향

- 지원자·학교·사원 JSON 구조는 변경하지 않았습니다.
- 기존 localStorage 핵심 키와 Supabase 테이블 구조는 변경하지 않았습니다.
- 별도 Supabase 마이그레이션은 필요하지 않습니다.

## 검증

- 실제 사원 조직정보 전용 JSON 1,381행을 백업센터에서 선택해 전용 파일로 판별하고 적용 버튼이 차단되는 것을 확인했습니다.
- ERP 전체 백업, 지원자 전용 배열, 사원 조직정보 전용 파일 유형 판별을 확인했습니다.
- 팀·그룹·제품·파트 연동 선택과 결과 필터링을 브라우저에서 확인했습니다.
- 검색 조건 저장·불러오기용 localStorage 저장을 확인했습니다.
- 지원자 목록의 `상세 / 수정 / 삭제` 버튼 회귀 검사를 통과했습니다.
- 데스크톱·모바일 렌더링, 전체 JavaScript 문법, 중복 HTML ID, 로컬 파일 참조를 검사했습니다.
- 브라우저 pageerror 및 console.error 0건을 확인했습니다.
