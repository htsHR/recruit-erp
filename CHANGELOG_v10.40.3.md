# Recruit ERP v10.40.3 EXCEL_PASTE_STABILIZATION

- 실제 `구직 지원자 명단 (2026)` 시트의 지원자 12개 행을 기준으로 열 매핑을 재검증했습니다.
- 기존 22열 형식과 향후 `생년월일` 열을 사용하는 2026 형식을 모두 지원합니다.
- 붙여넣은 원본 행을 열 이름과 함께 미리 볼 수 있습니다.
- 성명·지원일·연락처·지원근무지·출근방법 필수값과 전화번호·이메일·날짜 형식을 자동 검사합니다.
- 상태와 면접일·입사일, 날짜 순서가 맞지 않으면 입력폼 적용을 차단합니다.
- 지원근무지와 출근방법을 직접 확인해야 적용할 수 있습니다.
- 연락처·이메일·성명+생년월일 기준 중복 후보와 일치 사유를 표시하고, 확인 전에는 적용을 차단합니다.
- 적용 전 최종 요약을 표시하고 오류가 남아 있으면 적용 버튼을 비활성화합니다.
- 신규 등록과 기존 지원자 수정 비교 방식, localStorage 키, JSON 구조, Supabase 저장 로직은 변경하지 않았습니다.
