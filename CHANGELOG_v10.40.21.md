# Recruit ERP v10.40.21

**HOME_DEV · UI_REGRESSION_GUARD**

## 목적

v10.40.20 UI 시스템 통합 이후 새 기능을 추가하지 않고, 기존 화면과 주요 업무 흐름의 회귀 오류를 점검해 다음 개발 기준점을 확정했습니다.

## 확인한 업무 흐름

- 12개 주요 페이지 전환과 상단 제목
- 지원자 검색·상세·수정·신규 등록
- 엑셀 한 행 붙여넣기 모달
- 수동 일정 등록
- 안내문 생성
- 협력학교 검색·상세
- 사원 검색·상세·수정·상태 점검
- 데이터 점검센터 및 중복 후보 재계산
- ERP 백업·지원자 전용·사원 조직정보 JSON 유형 판별
- 회사 모드와 집 모드의 업로드 영역 분리
- 지원자 `상세 / 수정 / 삭제` 및 사원 `상세 / 수정` 버튼 보존

## 화면 회귀검사

- 1648×910 데스크톱
- 1366×768 노트북
- 390×844 모바일
- 지원자 목록, 사원명부, 협력학교, 데이터 점검, 중복관리, 백업센터의 가로 넘침 없음
- 72명 지원자·444개 학교·1,381명 사원 규모 렌더링 확인

## 정적 검사

- 전체 JavaScript 문법 검사 통과
- HTML 중복 ID 없음
- CSS·JS 로컬 파일 참조 누락 없음
- CSS 중괄호 구조 이상 없음
- ZIP 무결성 검사 통과
- 브라우저 pageerror 및 console.error 0건

## 데이터 영향

- 지원자·협력학교·사원·일정 데이터 구조 변경 없음
- localStorage 키 변경 없음
- Supabase 테이블 및 저장 로직 변경 없음
- 추가 SQL 없음

## 운영 메모

삼성 회사망 실기기 검증은 주말 동안 수행하지 않았습니다. 출근 후 회사 모드 자동 인식, Supabase 조회, 전체 JSON 다운로드를 짧게 확인하면 됩니다.
