- 상단 소개, 상태 카드, 점검 바, 보기 전환, 검색·필터 영역을 업무형 밀도로 재구성했습니다.
- 실제 1648×910 화면에서 사원 목록 헤더와 첫 행이 한 화면에 함께 보이도록 세로 공간을 줄였습니다.
- 전체·재직·휴직·퇴직·입사예정 카드는 클릭 즉시 상태 필터로 동작합니다.

- 목록은 사번 / 성명 / 소속 / 직급 / 입사일 / 근속기간 / 출신학교 / 재직상태 / 관리 구조를 유지합니다.
- 모바일에서는 상태 카드와 사원 목록이 카드형으로 재배치됩니다.

## 재직상태 관리

- 재직 → 휴직: 휴직 시작일 필수
- 휴직 → 재직: 복직일 필수
- 재직 → 퇴직: 퇴사일 필수
- 입사예정 → 재직: 실제 입사일 확인 및 미래 날짜 차단
- 입사일·휴직일·복직일·퇴사일의 순서 오류를 저장 전에 차단합니다.
- 휴직일 누락, 퇴사일 누락, 재직자의 퇴사일 존재, 입사예정일 경과, 사번 중복 등을 오류·주의·확인 필요로 분류합니다.
- 자동 수정은 하지 않으며 사용자 확인 후 상태와 날짜를 함께 반영합니다.

## 사원명부 엑셀 비교

- `.xlsx` 파일을 브라우저에서 직접 읽으며 재직자, 휴직자, 퇴직자 시트를 자동 구분합니다.
- 사번을 기준으로 신규 / 변경 / 동일 / ERP에만 존재 / 사번·이름 오류 / 상태 불일치를 분류합니다.
- 팀, 그룹, 제품, 파트, 직급, 직책, 입사일, 퇴사일, 재직상태, 출신학교, 전공을 비교합니다.
- 변경된 필드만 선택할 수 있으며 빈 엑셀 셀은 기존 값을 삭제하지 않습니다.
- 반영 직전 전체 ERP JSON 안전 백업을 자동 생성합니다.
- 선택 반영 결과를 요약하고 직전 반영 실행 취소를 지원합니다.
- 회사 모드에서는 파일 불러오기와 반영을 차단합니다.

## 데이터 영향

- 기존 지원자·학교·사원 JSON 구조와 localStorage 키를 변경하지 않습니다.
